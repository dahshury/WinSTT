### End-to-end flows

```mermaid
flowchart TD
  user[User clicks trigger / item / rail]
  trigger[Combobox.Trigger]
  root[Combobox.Root\nopen/onOpenChange]
  list[Combobox.List + Group + Item]
  selector[SttModelSelector\nopen=useState, lastClickTargetRef]
  filters[SttFiltersMenu\nPopover.Root]
  rail[GroupRail (DOM)]

  user --> trigger
  trigger -- press --> root
  root -- onOpenChange(true) --> selector
  selector -- setOpen(true) --> root
  root --> list
  list -- item-press --> root
  root -- onOpenChange(false, reason='item-press') --> selector
  selector -- applyCloseWith --> root

  user --> filters
  filters -- pointerdown captured by --> selector
  filters -- Popover open/close --> filters
  root -- outside-press (filter click) --> selector
  selector -- isInsideMenuPopup ⇒ veto --> root

  user --> rail
  rail -- scrollIntoView on data-rail-section --> list
```

1. **Rail-click "scroll to top" vs "no empty area" is a hard trade-off.** Without virtualization (#10), the only ways to scroll a *small* last group to viewport top are: (a) padding the list (user rejected: "huge area underneath"), (b) `min-height: 100cqh` on the last `Combobox.Group` (same visual outcome — empty area inside the group), or (c) a popup-height shrink on rail click (jarring).
