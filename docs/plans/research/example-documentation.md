# Example study — Feature-Sliced Design "documentation"

The FSD methodology docs repo (a docs site, not an STT app). Relevant as a **documentation
craft** reference (and WinSTT's frontend follows FSD).

## Stealable
- **"Advantages" section** — each advantage is a bold one-liner + a two-line explanation
  (Uniformity, Controlled reuse, Stability, Business-orientation). A crisp, scannable
  benefit list. WinSTT's "Why local-first" grid on `comparison.mdx` mirrors this shape.
- **Community badges** (Discord / Telegram / Website) aligned top-right of the banner, plus
  a "Show off" badge users can embed. WinSTT could add a community/Discussions badge + a
  "built with WinSTT" idea (improvement #11).
- **Methodology-first framing** — explains the *why* before the *rules*. Matches WinSTT's
  "benefit before mechanism" voice rule.
- Strong, consistent IA: Advantages → concepts → reference → guides — the progressive
  disclosure WinSTT's sidebar now follows (Getting Started → Using → Reference → Help).

## Avoid
- Badge/logo-heavy header that pushes content below the fold — keep WinSTT's hero tight.
