WinSTT supports 30+ first-party speech recognition models across six families — **whisper**, **lite-whisper**, **nemo**, **gigaam**, **kaldi**, and **t-one** — plus user-provided **custom** ONNX bundles dropped into the dedicated `models/custom/` folder.

## One ONNX Engine

All models — including Whisper — run on a single inference engine: ONNX Runtime via the `onnx-asr` library. There is no separate backend to choose. The legacy `backend` setting (`faster_whisper` / `onnx_asr`) is vestigial: `faster_whisper` is now an alias that routes to the same ONNX engine.

CPU vs GPU is decided **at install time**, not in settings:

- **CPU build** — bundles `onnxruntime`. Runs on any machine.
- **GPU build** — bundles `onnxruntime-gpu` plus the NVIDIA CUDA wheels. Uses the GPU when a working CUDA stack is present, and automatically falls back to CPU otherwise.

The `device` setting accepts `auto` or `cpu` only. `auto` picks the GPU when the GPU build is installed with working CUDA.

## Precision

Numerical precision is controlled by the `onnxQuantization` setting. An empty string (`""`) uses the model's default precision; other values (`fp16`, `q4`, `bnb4`, `int8`) are available per-model and surfaced by the model picker. See [Compute Types](/docs/models/compute-types).

## Custom Models

Drop a HuggingFace-style ONNX Whisper export into `{userData}/models/custom/{slug}/` and WinSTT will surface it in the picker on the next restart under the **Custom** group. The picker's "Open custom models folder" button takes you straight there — no need to hunt down the `%APPDATA%` path by hand.

### Folder contract

Each subfolder you drop becomes a single picker entry whose id is `custom-{slug}`. The slug is the folder name; we use the trailing path segment of `_name_or_path` from `config.json` for the display label when present, otherwise we title-case the slug.

A folder is registered as **valid** when it contains all of the following:

| File | Required | Notes |
|---|---|---|
| `encoder.onnx` **or** `encoder_model.onnx` | yes | Either filename is accepted (Optimum-style or onnx-community-style export). |
| `decoder_model.onnx` **or** `decoder_model_merged.onnx` | yes | Merged-decoder graphs are preferred. |
| `tokenizer.json` | yes | HuggingFace tokenizers JSON format. The legacy `vocab.json` + `merges.txt` pair is **not** accepted. |
| `config.json` | yes | Must be a JSON object containing a non-empty `model_type` string. The optional `_name_or_path` drives the display label. |
| `preprocessor_config.json` | no | Picked up automatically by onnx-asr if present. |
| `generation_config.json` | no | Picked up automatically by onnx-asr if present. |

### Broken drops

Folders that fail the contract still appear in the picker, greyed out, with a tooltip explaining what's missing (e.g. `Unavailable: missing tokenizer.json in my-export`). This makes a partial download / wrong export layout obvious at a glance, instead of leaving you wondering why your folder didn't show up.

### Caveats

- WinSTT does not auto-quantize custom models. Use whatever precision you exported — the picker only offers the default (un-suffixed) graph for `custom-*` entries.
- Custom models bypass the hardware-fit check because we don't know the parameter count without loading the weights. If your model exceeds the GPU's VRAM, ONNX Runtime will fail to create the session at load time.
- Custom models are local-only — they do not appear in `list_models_with_state`'s download/cache metadata because they are already on disk.
