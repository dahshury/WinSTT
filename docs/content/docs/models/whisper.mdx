Whisper is OpenAI's general-purpose speech recognition model, and the Whisper family is the most flexible engine WinSTT ships: 13 entries spanning a 38M-parameter realtime model up to a 1.55B-parameter accuracy ceiling, covering 99 languages with optional one-pass translation to English.

WinSTT runs every Whisper model through the same `onnx-asr` ONNX Runtime engine as all other families — **not** CTranslate2/faster-whisper. There is no backend to choose; pick a model in the [Model](/docs/settings/model) tab and it loads on the one engine.

## How to read the numbers

Every model row below carries four catalog figures. They are the same numbers the model picker shows as accuracy/speed bars.

<ModelTable
  head={["Field", "Means", "Direction"]}
  rows={[
    ["Params", "Parameter count — drives RAM/VRAM and download size.", "Lower = lighter"],
    ["WER", "Word error rate on a mixed multilingual benchmark, as a percentage.", "Lower = more accurate"],
    ["RTFx", "Real-time factor — how many seconds of audio it transcribes per wall-clock second.", "Higher = faster"],
    ["Langs", "Number of languages the model can transcribe.", "Higher = broader"],
  ]}
  caption="RTFx is engine-and-hardware dependent; treat it as a relative ranking, not an absolute throughput guarantee."
/>

All Whisper-family models support the realtime live preview, so any of them can serve as the realtime model as well as the main model.

## English-only models

## Lite-Whisper

Lite-Whisper is Efficient-Speech's structurally compressed `large-v3-turbo`. It keeps the full 99-language multilingual coverage but with a lower-rank encoder, so it loads smaller and runs faster than turbo at a modest WER cost. Three points on the size/speed curve:

<ModelTable
  head={["Model", "Params", "WER", "RTFx", "Trade-off"]}
  numeric={[1, 2, 3]}
  rows={[
    ["lite-whisper-large-v3-turbo-acc", "581M", "8.0", "320", "Most accuracy retained"],
    ["lite-whisper-large-v3-turbo", "534M", "8.5", "380", "Balanced compression"],
    ["lite-whisper-large-v3-turbo-fast", "474M", "9.5", "500", "Most aggressive compression"],
  ]}
  caption="All three are 99-language multilingual and support realtime."
/>

<Callout type="warn" title="The -acc suffix means accelerated, not accurate">
  Counter-intuitively, `lite-whisper-large-v3-turbo-acc` is the *least* compressed of the three — it keeps the most parameters and therefore the most accuracy. The plain (no-suffix) variant is the balanced middle, and `-fast` is the smallest and quickest. Read by the numbers above, not the suffix.
</Callout>

## Fine-tunes

Two specialist Whisper-large fine-tunes for narrower jobs. Both are 1.5B-class, multilingual-capable, and ship in default precision only.

<ModelTable
  head={["Model", "Params", "WER", "RTFx", "Langs", "Specialty"]}
  numeric={[1, 2, 3, 4]}
  rows={[
    ["crisper-whisper", "1.54B", "6.7", "150", "2", "Verbatim transcription — keeps fillers (um/uh), stutters, and false starts instead of cleaning them up. English + German."],
    ["breeze-asr-25", "1.55B", "8.0", "150", "99", "MediaTek Breeze — a large-v2 fine-tune tuned for Mandarin/English code-switching."],
  ]}
  caption="CrisperWhisper is a large-v3 fine-tune (nyrahealth); Breeze ASR 25 is a large-v2 fine-tune that retains the full multilingual vocabulary."
/>

<Callout type="info" title="Verbatim vs. clean">
  Most ASR models silently delete disfluencies. **CrisperWhisper** does the opposite — it transcribes exactly what was said, fillers and all. Use it for interview/legal transcripts where every "um" matters; avoid it for dictation, where you want the clean version.
</Callout>

## Translate to English

Multilingual Whisper models support OpenAI's native `task=translate`. Turn on **Translate to English** in the [Model](/docs/settings/model) tab and the model transcribes **and** translates to English in a single decode pass — no extra latency and no LLM round-trip.

<ModelTable
  head={["Model class", "Translate to English", "Note"]}
  rows={[
    ["Multilingual (tiny … large-v3, lite-whisper-*, breeze-asr-25)", "Supported", "Single-pass; toggle in the Model tab."],
    ["English-only (.en)", "N/A", "Source is already English."],
    ["crisper-whisper (en/de)", "Supported", "German source → English output."],
  ]}
  caption="Translate is a Whisper-native task; non-Whisper families translate via the LLM cleanup pass instead."
/>

## Precision (quantizations)

Whisper precision is set per-model via the picker's quantization badge (see [Compute Types](/docs/models/compute-types)). The empty default (`""`) is fp32 — maximum accuracy. The picker only offers what the upstream repo actually publishes.

<ModelTable
  head={["Models", "Available quantizations"]}
  rows={[
    ["tiny, base, small, medium, large-v3-turbo, and all .en variants", "default · fp16 · q4 · bnb4"],
    ["lite-whisper-large-v3-turbo / -acc / -fast", "default · fp16"],
    ["large-v3, crisper-whisper, breeze-asr-25", "default only"],
  ]}
  caption="fp16 roughly halves the download and is the fastest path on DirectML GPUs; q4 / bnb4 cut size to ~13% for the smallest footprint. The server falls back to default precision if a chosen variant is missing."
/>

<Callout type="restart">
  Switching between Whisper models hot-swaps live without a restart. Changing to a different model *family* (e.g. Whisper → NeMo) reconnects the engine — the overlay shows a brief "swapping" state.
</Callout>

## When to use which

<BentoGrid cols={3}>
  <BentoCell icon="speed" title="Realtime preview">
    <strong>tiny</strong> or <strong>base</strong> (or <strong>.en</strong> for English) — 510–750 RTFx keeps the live preview snappy.
  </BentoCell>
  <BentoCell icon="quality" title="Best all-round main model">
    <strong>large-v3-turbo</strong> — 7.8 WER at 250 RTFx and a 1.13 GB download. The default recommendation.
  </BentoCell>
  <BentoCell icon="brain" title="Maximum accuracy">
    <strong>large-v3</strong> — 7.4 WER, the family's accuracy ceiling, at the cost of a 4.54 GB download and the lowest RTFx.
  </BentoCell>
  <BentoCell icon="cpu" title="Smaller GPU / faster turbo">
    <strong>lite-whisper-large-v3-turbo-acc</strong> — 8.0 WER, 581M params, full multilingual coverage.
  </BentoCell>
  <BentoCell icon="languages" title="English-only boost">
    The <strong>.en</strong> variants shave 1–2 WER off English audio versus their multilingual twins.
  </BentoCell>
  <BentoCell icon="snippets" title="Verbatim transcripts">
    <strong>crisper-whisper</strong> — keeps every filler and false start for interview and legal work.
  </BentoCell>
</BentoGrid>

<Callout type="info" title="Recommended pairing">
  Run <strong>large-v3-turbo</strong> as the main model for the final, accurate paste, and pair it with <strong>tiny</strong> or <strong>base</strong> as the realtime model for a responsive live preview. If your main model is already small enough to run live, WinSTT locks one model into both slots automatically.
</Callout>
