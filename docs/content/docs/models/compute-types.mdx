---
title: Compute Types
description: ONNX quantization options for model inference
---

Quantization controls the numerical precision used during model inference. Lower precision reduces memory usage and increases speed at the cost of minor quality differences. WinSTT runs every model on ONNX Runtime, so precision is set with the `onnxQuantization` setting.

An empty string (`""`) uses the model's default precision. Other values are available per-model and surfaced by the model picker — selecting an unsupported value for a given model is not offered.

## Available Quantizations

| Value | Description | Where available |
|-------|------------|-----------------|
| `""` | Model default precision | Every model |
| `fp16` | Half 16-bit precision | Most Whisper, Lite-Whisper |
| `q4` | 4-bit weight quantization | Most Whisper |
| `bnb4` | bitsandbytes 4-bit quantization | Most Whisper |
| `int8` | 8-bit integer quantization | NeMo, GigaAM, Vosk/Kaldi |

`large-v3` and `t-tech/t-one` support default precision (`""`) only.

## Recommendations

<Callout title="GPU users" type="info">
  On the GPU build with a working NVIDIA CUDA stack, `fp16` cuts memory roughly in half versus full precision with negligible quality loss, and modern NVIDIA GPUs accelerate it.
</Callout>

<Callout title="CPU users" type="info">
  For NeMo, GigaAM, and Vosk/Kaldi on CPU, `int8` gives the best performance with minimal quality loss. For Whisper, `q4` or `bnb4` reduce memory footprint when RAM is tight. Use `""` when unsure or when you want maximum quality.
</Callout>

## CPU vs GPU

CPU vs GPU is chosen **at install time**, not in settings — WinSTT ships as two installers (a CPU build and a GPU build). The `device` setting accepts only `auto` or `cpu`; `auto` uses the GPU when the GPU build is installed with working CUDA, and falls back to CPU otherwise.

## How to Set

Configure `onnxQuantization` in [Model Settings](/docs/settings/model). The setting applies to both the main model and the realtime model.

<Callout title="Legacy setting" type="warn">
  The old `computeType` setting (CTranslate2-style float/int compute types) is a no-op on the ONNX path. It is retained only for backward compatibility and has no effect on inference.
</Callout>
