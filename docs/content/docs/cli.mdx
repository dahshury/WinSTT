---
title: Command-Line Flags
description: The flags the WinSTT app accepts at launch, and the developer-only spike binaries for benchmarking STT/TTS.
---

WinSTT is a single native executable. It accepts a small set of launch flags, and because it
runs single-instance, three of them are **forwarded to an already-running instance** — which
makes them useful targets for OS-level keyboard shortcuts or automation.

```bash
WinSTT [OPTIONS]
```

<StatGrid>
  <Stat value="1" label="Native executable" />
  <Stat value="7" label="Launch flags" />
  <Stat value="3" label="Forwarded to the running instance" />
</StatGrid>

<Callout type="info" title="Most config lives in the UI, not the CLI">
  The model, language, device, VAD, realtime, wake words, and TTS are all configured in the
  [Settings UI](/docs/settings) and persisted. The flags below only control startup behavior and
  the three forwarded actions you can wire to a shortcut.
</Callout>

## App flags

<ModelTable
  head={["Flag", "What it does"]}
  rows={[
    ["--start-hidden", "Start with the main pill window hidden (tray only)."],
    ["--no-tray", "Don't create the system-tray icon."],
    ["--toggle-transcription", "Toggle dictation on/off. Forwarded to the running instance."],
    ["--toggle-post-process", "Toggle dictation with LLM post-processing on/off. Forwarded to the running instance."],
    ["--cancel", "Cancel the current operation. Forwarded to the running instance."],
    ["--debug", "Enable verbose logging."],
    ["-h, --help", "Print help and exit."],
  ]}
/>

<Callout type="info" title="The forwarded flags drive shortcuts">
  WinSTT runs single-instance. Launching the exe again with `--toggle-transcription`,
  `--toggle-post-process`, or `--cancel` doesn't open a second window — it hands the flag to the
  instance that's already running. Bind a global shortcut (or a stream-deck button, or a script)
  to `WinSTT.exe --toggle-transcription` to start/stop dictation from anywhere.
</Callout>

## Developer spike binaries

<Callout type="warn" title="Developer-only — not in the shipped app">
  These `cargo run --example` harnesses live in `src-tauri/examples/` and are driven by
  environment variables. You only need them if you're hacking on the inference path. They are
  used to benchmark and de-risk the STT/TTS engines during the port.
</Callout>

### `stt_spike` — STT decode benchmark

Decodes a fixed 16 kHz JFK sample through the unified `ort` engine and prints the transcript +
timing, so you can A/B a model, quantization, or execution provider.

```bash
cargo run --release --example stt_spike                       # whisper-tiny.en, CPU
cargo run --release --example stt_spike -- <hf_snapshot_dir> [n_mels] [lang]
SPIKE_PROVIDER=dml cargo run --release --example stt_spike    # measure the DirectML path
```

<ModelTable
  head={["Env var", "What it does"]}
  rows={[
    ["SPIKE_PROVIDER", "Execution provider: cpu (default), dml, or cuda."],
    ["SPIKE_QUANT", "Override the resolved quantization (e.g. int8, fp16) to A/B precisions."],
    ["SPIKE_CACHE_ONLY", "Resolve from the on-disk HF cache only (no network)."],
    ["HF_HUB_CACHE / HF_HOME", "Where to find / download model snapshots."],
  ]}
/>

### `tts_spike` — Kokoro TTS benchmark

Times the shipped Kokoro synthesis path (warm/cold) on CPU or DirectML.

```bash
cargo run --release --example tts_spike
SPIKE_TTS_DEVICE=dml cargo run --release --example tts_spike   # measure the GPU path
```

<ModelTable
  head={["Env var", "What it does"]}
  rows={[
    ["SPIKE_TTS_DEVICE", "Execution device: cpu (default) or dml."],
    ["WINSTT_KOKORO_DIR", "Override the model / voicepack location."],
  ]}
/>

### `tts_engine_spike` — exploratory TTS engines

Runs the experimental TTS engines (kitten / piper / supertonic / chatterbox) that are **not**
in the shipped app, loading their weights from `.tts-cache/<engine>/`.

```bat
rem from the repo root — wraps the MSVC env + redirects output to spike_out.log
tools\windows\spike.bat <engine> [voice] [text]
```

<ModelTable
  head={["Env var", "What it does"]}
  rows={[
    ["WINSTT_TTS_CACHE", "Override the cache root that holds .tts-cache/<engine>/."],
    ["ESPEAK_NG_LIBRARY", "Path to the espeak-ng DLL (auto-resolved to the bundled copy if unset)."],
  ]}
/>
