title: Dictionary & Snippets
description: Fuzzy term correction and text expansion
---

WinSTT provides two text transformation features that run on every transcription output: dictionary corrections and snippet expansions.

## Dictionary

A dictionary entry is a single canonical word, name, or phrase you want WinSTT to get right. There is no find/replace, no case-sensitive or whole-word toggle — just the canonical `term`.

At post-process time the server fuzzy-matches every transcribed token against each dictionary term using **Jaro-Winkler + Double Metaphone** similarity, and replaces the closest near-miss with the canonical `term`. Use it for proper nouns, product names, and jargon the model tends to mishear.

<Callout title="Migration" type="warn">
  Dictionary entries are now `{ id, term }`. The old find/replace/caseSensitive/wholeWord model has been removed and a store migration clears any pre-existing dictionary entries.
</Callout>

### Entry Fields

| Field | Type | Description |
|-------|------|-------------|
| `id` | string | Stable identifier (assigned automatically) |
| `term` | string | The canonical word, name, or phrase |

### Examples

| Term | Effect |
|------|--------|
| `Kubernetes` | "cooper netties" / "kubernets" is corrected to `Kubernetes` |
| `PyTorch` | "pie torch" is corrected to `PyTorch` |
| `Anthropic` | "anthropik" is corrected to `Anthropic` |
| `Khaled` | A name the model mishears is normalized to `Khaled` |

## Snippets

Snippets are literal text expansions. The `trigger` string is replaced verbatim with `expansion` in the output. Use them for inserting predefined blocks of text.

### Entry Fields

| Field | Type | Description |
|-------|------|-------------|
| `id` | string | Stable identifier (assigned automatically) |
| `trigger` | string | Short text that activates the expansion |
| `expansion` | string | Full text inserted in place of the trigger |

### Examples

| Trigger | Expansion |
|---------|-----------|
| `addr` | `123 Main St, City, ST 12345` |
| `sig` | `Best regards, John Smith` |
| `eml` | `john.smith@example.com` |
