---
title: Recording Modes
description: Push-to-Talk, Toggle, Listen, and Wake Word modes
---

WinSTT supports four recording modes. Set the mode in [General Settings](/docs/settings/general) using the `recordingMode` field (`ptt`, `toggle`, `listen`, `wakeword`).

## Push-to-Talk (PTT)

**Default mode.** Hold the hotkey to record. Release to stop recording and trigger transcription.

1. Press and hold the hotkey. Recording starts immediately.
2. Speak while holding the key.
3. Release the hotkey. Recording stops and the audio is sent for transcription.

Voice activity detection is not used for stop detection in PTT mode. The duration is controlled entirely by how long you hold the key — release triggers a fast stop.

**Best for:** Dictation bursts, quick voice commands, noisy environments where VAD might false-trigger.

## Toggle Mode

Press the hotkey once to start listening. Voice activity detection handles the rest. Press again to exit.

1. Press the hotkey. WinSTT enters listening mode and waits for speech.
2. Start speaking. VAD detects your voice and begins recording.
3. Stop speaking. After `postSpeechSilenceDuration` of silence, recording stops and transcription begins.
4. If you start speaking again, a new recording begins automatically.
5. Press the hotkey again to exit listening mode entirely.

Set `manualToggleStop` (boolean) in General Settings to record continuously from the first press to the second press. With it on, VAD silence-stop and silence-timing punctuation tuning are disabled — good for long-form dictation with natural pauses.

**Best for:** Hands-free dictation, long passages, transcribing meetings or lectures.

## Listen Mode

Passively captures from a loopback / system-audio device — no hotkey involved. It transcribes continuously, gated by VAD.

Set `loopbackDeviceIndex` in General Settings, and install a loopback driver such as VB-Audio Virtual Cable so the device is available. `systemAudioReductionWhileDictating` is ignored in this mode.

**Best for:** Transcribing meetings, calls, and other system audio.

## Wake Word Mode

Continuously listens for a Porcupine built-in keyword. No hotkey is used.

1. WinSTT waits for the configured `wakeWord` (e.g. `alexa`, `computer`, `jarvis`, `hey google`).
2. On detection it starts recording.
3. It ends the turn via VAD-based silence detection (same as Toggle).
4. The detector resumes waiting for the wake word.

Tuning (General Settings): `wakeWordSensitivity` 0–1 (default 0.6), `wakeWordTimeout` 1–30s (default 5).

**Best for:** Fully hands-free, always-on voice activation.

## Smart Endpoint

Optional and cross-cutting. When `smartEndpoint` is on, a DistilBERT classifier decides when an utterance is complete instead of fixed silence timers, and `smartEndpointSpeed` tunes how eagerly it finalizes. It requires the server `sentence-classifier` extra and fails soft if that extra is absent.

## Silence Duration Behavior

The `postSpeechSilenceDuration` setting (in [Audio Settings](/docs/settings/audio)) behaves differently per mode:

| Mode | Silence Duration Role |
|------|----------------------|
| Push-to-Talk | Not used for stop. Release triggers a fast stop |
| Toggle | Active — controls how long silence must elapse before the turn ends. Disabled when `manualToggleStop` is on |
| Listen | Active — VAD-gated continuous capture uses it to segment turns |
| Wake Word | Active after detection — ends the turn the same way as Toggle |

Lower values make WinSTT more responsive to pauses but may cut off speech with natural breaks. Higher values tolerate longer pauses but delay the final transcription. When Smart Endpoint is on, the classifier supersedes these fixed timers.
