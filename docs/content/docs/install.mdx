WinSTT ships as a self-contained portable `.exe`: download one file, run it, and dictate. There is no installer wizard, no Python, no model download required to start.

## Which installer?

Every release publishes three portable builds on the [GitHub Releases](https://github.com/dahshury/WinSTT/releases) page. All three wrap the **same** Tauri app and Rust speech engine — only the bundled ONNX Runtime (ORT) execution provider differs. Pick by your hardware:

  head={["Installer", "Size", "ORT wheel", "When to use"]}
  numeric={[1]}
  rows={[
    ["WinSTT-Portable-<ver>.exe", "~200 MB", "onnxruntime-directml", "Default GPU build. Any D3D12-capable GPU — AMD, Intel, or NVIDIA via DirectX 12. The unmarked default download."],
    ["WinSTT-CPU-Portable-<ver>.exe", "~150 MB", "onnxruntime", "No GPU, a headless box, or you just want the smallest download. CPU-only."],
    ["WinSTT-OpenVINO-Portable-<ver>.exe", "~250 MB", "onnxruntime-openvino", "Intel ARC dGPU, or recent Iris Xe / Arc iGPU. ~10-30% faster than DirectML on Intel silicon (Intel-published benchmarks)."],
  ]}
  caption="The <ver> in each filename is the release tag (e.g. WinSTT-Portable-0.X.0.exe)."
/>

<Callout type="info" title="Not sure? Take DirectML.">
  The default **WinSTT-Portable** (DirectML) build runs on every modern Windows GPU and
  automatically falls back to CPU when no compatible GPU is present — so it's safe to
  install anywhere. Only choose CPU if you specifically want the smallest download, or
  OpenVINO if you're on Intel ARC / Iris Xe graphics.
</Callout>

No CUDA installer is shipped on Windows. On our workload DirectML is faster and ~10× lighter than CUDA — on an RTX 3080 Ti, DirectML runs whisper-tiny-q4 at **85 ms p50** versus CUDA's **120 ms p50**, while the installer is ~200 MB instead of ~2 GB. The CUDA path is reserved for a future Linux NVIDIA build.

## Auto-fallback to CPU

The GPU builds degrade gracefully. At startup the Rust backend probes the active execution provider, and if the requested GPU path isn't viable — no D3D12 GPU for DirectML, no Intel accelerator for OpenVINO — it silently falls back to CPU. You get a working app either way, so installing the DirectML build on a GPU-less machine is harmless.

<Callout type="info" title="Force CPU manually">
  You can pin CPU regardless of the installed build from the **Model** tab's Device
  control (set it to **CPU**), which is useful on machines where the GPU driver is flaky.
</Callout>

<SettingRow label="Microphone" settingKey="">
  Any input device PortAudio can open. For **Listen** mode you also need a loopback /
  monitor device (e.g. VB-Audio Virtual Cable) so the app can capture system audio.
</SettingRow>

<Callout type="success" title="No Python, no accounts, no telemetry">
  End users never install Python, uv, or any model files. STT and TTS run inside the
  bundled Rust/Tauri executable. Models download on demand from the UI. Transcription
  runs entirely on your machine; there are no usage analytics. (Anonymized crash reports via
  Sentry are opt-out.)
</Callout>

## Install steps

<StepFlow>
  <Step title="Download the installer">
    Grab the latest default Windows build:
    [WinSTT.exe](https://github.com/dahshury/WinSTT/releases/latest/download/WinSTT.exe).
    For alternate builds and signatures, use the
    [GitHub Releases](https://github.com/dahshury/WinSTT/releases) page.
  </Step>
  <Step title="Verify the signature (recommended)">
    Each `.exe` ships a [minisign](/docs/verify-releases) `.minisig` sidecar so you can
    confirm the download offline, on top of the Authenticode signature Windows already
    checks. See [Verify Release Signatures](/docs/verify-releases).
  </Step>
  <Step title="Run it">
    Double-click the `.exe`. It's portable — no install wizard, no admin rights needed for a
    normal launch. WinSTT starts in the system tray.
  </Step>
  <Step title="Onboard & pick a model">
    On first launch you'll choose a microphone, a hotkey, and a model. The default model
    downloads in the background; you can switch models any time from the **Model** tab
    without a restart.
  </Step>
</StepFlow>

<Callout type="warn" title="SmartScreen on first run">
  Windows SmartScreen may warn on a freshly released, low-reputation binary. Click
  **More info → Run anyway** after you've verified the signature. The installers are
  Authenticode-signed, and reputation builds up as more people download each release.
</Callout>
