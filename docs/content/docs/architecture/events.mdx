---
title: Domain Events
description: Server-side events published via EventBus
---

The WinSTT server uses an event-driven architecture. All domain events are frozen dataclasses that inherit from `RecorderEvent` and carry a `timestamp` field. Events are published through a thread-safe `EventBus` using pub/sub.

Legacy callbacks (e.g., `on_recording_start`) are bridged to these events in `bootstrap.py` via `wire_callback()`. Application code subscribes to events rather than using raw callbacks.

## Event Reference

### Recording Lifecycle

| Event | Data Fields | Description |
|-------|------------|-------------|
| `RecordingStarted` | `timestamp` | Audio recording began |
| `RecordingStopped` | `timestamp` | Audio recording ended |
| `TranscriptionStarted` | `timestamp`, `audio` | Full transcription started. `audio` contains the raw bytes |
| `TranscriptionCompleted` | `timestamp`, `text` | Full transcription finished. `text` contains the result |

### Voice Activity Detection

| Event | Data Fields | Description |
|-------|------------|-------------|
| `VADStarted` | `timestamp` | Voice activity detection system enabled |
| `VADStopped` | `timestamp` | Voice activity detection system disabled |
| `VADDetectStarted` | `timestamp` | Speech onset detected (voice became active) |
| `VADDetectStopped` | `timestamp` | Speech offset detected (voice went silent) |

### Turn Detection

| Event | Data Fields | Description |
|-------|------------|-------------|
| `TurnDetectionStarted` | `timestamp` | Turn detection enabled (listening for end of speech) |
| `TurnDetectionStopped` | `timestamp` | Turn detection disabled |

### Wake Word

| Event | Data Fields | Description |
|-------|------------|-------------|
| `WakeWordDetected` | `timestamp`, `word_index`, `word` | A wake word was matched. `word_index` identifies which word in the list, `word` is the matched string |
| `WakeWordTimeout` | `timestamp` | Wake word detection timed out without a match |
| `WakeWordDetectionStarted` | `timestamp` | Wake word detector enabled |
| `WakeWordDetectionEnded` | `timestamp` | Wake word detector disabled |

### Realtime Transcription

| Event | Data Fields | Description |
|-------|------------|-------------|
| `RealtimeTranscriptionUpdate` | `timestamp`, `text` | Live preview text updated with latest partial transcription |
| `RealtimeTranscriptionStabilized` | `timestamp`, `text` | Live preview stabilized (text is unlikely to change further) |

### Audio

| Event | Data Fields | Description |
|-------|------------|-------------|
| `AudioChunkRecorded` | `timestamp`, `chunk` | A raw audio chunk was captured from the microphone. `chunk` contains PCM bytes |

### Model Downloads

| Event | Data Fields | Description |
|-------|------------|-------------|
| `ModelDownloadStarted` | `timestamp`, `model` | Model download began. `model` is the model identifier |
| `ModelDownloadProgress` | `timestamp`, `model`, `progress` | Download progress updated. `progress` is a float from 0.0 to 1.0 |
| `ModelDownloadCompleted` | `timestamp`, `model` | Model download finished successfully |

### Speaker Diarization

Emitted by the ONNX diarizer (`onnxasr_diarizer.py`) when speaker
attribution is enabled. These events carry the `timestamp` field plus a
diarization payload describing the detected speaker segments (a list of
segments, each with timing and a speaker label). Field names beyond
`timestamp` are payload-dependent.

### VAD Adaptation

The server automatically adapts the Silero VAD sensitivity to the current
background-noise level. When it adjusts, it publishes a VAD-adaptation event
carrying `timestamp` plus the new sensitivity value, so consumers can reflect
the change in diagnostics or UI.
