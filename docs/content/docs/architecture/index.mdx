title: Architecture Overview
description: How WinSTT is built
---

WinSTT is a two-process desktop application: a Python STT server and an Electron desktop client. They communicate over dual WebSocket channels -- one for JSON control messages and one for binary audio data.

**Server** -- Python, hexagonal architecture (ports and adapters). Handles audio processing, voice activity detection, and model inference.

**Frontend** -- Electron + Next.js 16, Feature-Sliced Design. Provides the user interface, manages the Python process lifecycle, and relays audio/events between the renderer and server.

The renderer process has zero direct WebSocket code. All server communication flows through the Electron main process via IPC.
