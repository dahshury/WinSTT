## Controls

<SettingRow label="Enable Text-to-Speech" settingKey="tts.enabled" default="false">
  Master on/off gate. Turning it on probes whether the voice pack is already cached; if
  not, you're prompted to download it (see below). The toggle is disabled while a download
  is in flight.
</SettingRow>

<SettingRow label="Voice" settingKey="tts.voice" default="af_heart">
  A searchable list of 54 voices grouped by language, each with a per-row preview button and
  a region badge (US, UK, JP, ZH). The languages in this list — US/UK English, Japanese,
  Mandarin, Spanish, French, Hindi, Italian, Brazilian Portuguese — are the only ones local
  TTS can speak. Picking a voice auto-updates the language. Default is **Heart** (US female,
  `af_heart`). If a saved voice ID is missing from the catalog, the first available voice is
  selected.
</SettingRow>

<SettingRow label="Speed" settingKey="tts.speed" default="1.0">
  Synthesis playback speed, **0.5x to 2.0x** in 0.1 steps, shown as `1.0x`. Applied by the
  Kokoro session itself (not post-processing), so pitch stays natural. A reset button
  appears when the value is non-default.
</SettingRow>

<SettingRow label="Speak hotkey" settingKey="tts.hotkey" default="(unset)">
  A uiohook key combo that reads the current text selection aloud. Set it from the
  [Hotkeys](/docs/settings/general) capture field; leave it empty to trigger TTS only from
  the tray or the preview buttons.
</SettingRow>

<Callout type="info" title="9 languages, local only">
  The voice picker is the full list of languages local TTS supports — Kokoro is the only
  bundled engine and it doesn't translate. ElevenLabs (already used for
  [cloud STT](/docs/settings/integrations)) covers far more languages, but **cloud
  text-to-speech isn't available in WinSTT yet**.
</Callout>

## On-demand voice pack

The Kokoro assets ship **separately** from the installer and download once on first enable,
then cache indefinitely. Total is **~190 MB**, split across three parts:

<ModelTable
  head={["Part", "Size"]}
  numeric={[1]}
  rows={[
    ["Engine pack", "~50 MB"],
    ["Kokoro model", "~100 MB"],
    ["Voicepacks", "~40 MB"],
    [<strong>Total</strong>, <strong>~190 MB</strong>],
  ]}
  caption="The voice pack downloads once on first enable, then caches indefinitely."
/>

An install banner under the controls shows phase and percent while the pack downloads,
with three controls:

<ModelTable
  head={["Control", "What it does"]}
  rows={[
    ["Pause", "Halt the download in place — the partial file is kept so you can resume later."],
    ["Resume", "Continue a paused download from where it stopped."],
    ["Cancel", "Abort and discard the in-flight download."],
  ]}
  caption="The Enable toggle stays disabled while a download is in flight."
/>

<Callout type="info" title="Downloaded once">
  Audio is synthesized fully on-device — nothing is sent to the cloud. After the first
  download the pack is reused on every launch, so subsequent enables are instant.
</Callout>

<Callout type="info" title="Device follows the Model tab">
  Text-to-Speech runs on the **same compute device as your speech-to-text model** — there's
  no separate TTS device picker. Set it once with **Device** in the
  [Model](/docs/settings/model) tab (`model.device`); the spawn layer mirrors that choice
  onto the server's `--tts-device` flag, so **Auto** runs Kokoro on the GPU (DirectML) where
  available and falls back to CPU otherwise.
</Callout>
