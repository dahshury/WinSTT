The **Model** tab is where you pick the engine that turns your speech into text. It also
configures the smaller model used for the live preview.

## Main model

The main model runs the **final, accurate** transcription pass — the text that actually
gets pasted. Everything in this section configures it.

<Video src="model-picker-flow" chrome="card" alt="The model picker opens with model rows, quantization badges, and a download progress bar moving in the background." caption="Search the catalog, pick a quantization, and let model downloads stream in the background." />

<Screenshot src="sec-model-main" alt="The Main Model section: a Local/Cloud source toggle, the model selector, language, device, model-unload timeout, and a Translate to English switch." chrome="card" caption="The Main Model section controls every part of the final transcription pass." />

<SettingRow label="Source" settingKey="model.model" default="Local">
  Run transcription **locally** (on-device ONNX models) or via a **cloud** provider
  (OpenAI Whisper, ElevenLabs). Cloud is locked until you add an API key in
  [Integrations](/docs/settings/integrations). Removing a cloud key silently reverts to
  your last local model.
</SettingRow>

<SettingRow label="Model" settingKey="model.model" default="tiny">
  A searchable, filterable list of 40+ models grouped by family. Each row shows its
  accuracy/speed bars, size, language scope, and per-quantization download badges.
</SettingRow>

<SettingRow label="Language" settingKey="model.language" default="en">
  Pin a language or leave it on auto-detect. Hidden entirely for single-language models.
</SettingRow>

<SettingRow label="Device" settingKey="model.device" default="auto">
  **Auto** uses the GPU when available and falls back to CPU; **CPU** forces CPU-only.
  Only CPU is offered on machines without a supported GPU.
</SettingRow>

<SettingRow label="Model Unload Timeout" settingKey="model.modelUnloadTimeout" default="min5">
  How long the server keeps the model resident after the last transcription before
  freeing RAM/VRAM. Options run from **Immediately** to **Never**.
</SettingRow>

<SettingRow label="Translate to English" settingKey="model.translateToEnglish" default="false">
  Uses Whisper's native `task=translate` to transcribe **and** translate to English in a
  single pass — no extra latency, no LLM round-trip. Only shown for multilingual Whisper
  (and Canary) models that support the translate task.
</SettingRow>

### The model picker

Opening the selector reveals the full catalog. Models are grouped by maker, each row
showing its hardware fit, size, the languages it covers, and which quantizations you can
download.

<Screenshot src="model-dropdown" alt="The open model picker dropdown: a maker rail on the left and model rows showing accuracy and speed bars, size, language scope, and quantization badges." label="Model picker" caption="Search, filter by family or language, and download a specific quantization per model." />

Each model exposes one or more **quantizations** — compressed weight formats that trade a
little accuracy for big wins in size and speed.

<ModelTable
  head={["Quantization", "Precision", "Relative size", "Best for"]}

    ["q4 / bnb4", "4-bit", "~13%", "Smallest download, lowest RAM"],
    ["q4f16", "mixed 4-/16-bit", "~20%", "Small with better quality than q4"],
  ]}
  caption="The picker only shows quantizations the upstream repo actually publishes; the server falls back to fp32 if a chosen variant is missing."
/>

<Callout type="info" title="Download, pause, resume">
  Downloads stream in the background without interrupting the running model, so the app
  stays connected. You can **pause**, **resume**, or **cancel** any download from its
  badge, and **delete** a single quantization to reclaim disk without removing the others.
</Callout>

## Realtime model

The realtime model powers the **live preview** that appears while you speak. It runs on a
smaller, faster model so words show up instantly; the main model still produces the final
text.

<Screenshot src="section-realtime" alt="The Realtime Model section with a model selector and an update-interval stepper." chrome="card" />

<SettingRow label="Realtime Model" settingKey="model.realtimeModel" default="tiny">
  Any model under ~700M parameters. If your **main** model is already small enough to run
  live, this slot is locked to it (one model serves both) and the picker is disabled.
</SettingRow>

<SettingRow label="Update Interval" settingKey="quality.realtimeProcessingPause" default="0.02">
  Minimum seconds between live-preview updates. Lower feels snappier but costs more compute.
</SettingRow>

<Callout type="restart">
  Most model changes hot-swap live. Switching the **backend** family or toggling cloud vs
  local reconnects the engine, which takes a moment — the overlay shows a brief "swapping"
  state.
</Callout>
