---
title: Model Settings
description: Configure transcription models and inference parameters
---

Model settings control which speech recognition models WinSTT uses and how they run. WinSTT uses two models simultaneously: a main model for accurate final transcription, and a realtime model for fast live preview.

The transcription stack is ONNX-only. There is effectively one engine — the `backend` value `faster_whisper` is a legacy alias that bootstrap routes to ONNX.

## Settings Reference

| Setting | Type | Default | Description |
|---------|------|---------|-------------|
| `model` | string | `"large-v2"` | Main transcription model used for final output |
| `realtimeModel` | string | `"tiny"` | Fast model used for live preview while you speak |
| `language` | string | `"en"` | Transcription language code |
| `onnxQuantization` | string | `""` | Precision selector for the ONNX path. `""` = the model's default precision |
| `computeType` | enum | `"default"` | Legacy CTranslate2-era knob. No effect on the ONNX path; kept for settings/IPC stability |
| `device` | enum | `"auto"` | `"auto"` uses the GPU when the GPU build + a working CUDA stack are present, otherwise CPU. `"cpu"` forces CPU |
| `backend` | enum | `"faster_whisper"` | `"faster_whisper"` or `"onnx_asr"`. Both resolve to the ONNX engine; effectively vestigial |
| `beamSize` | integer | `5` | Beam search width for the main model (min 1) |
| `beamSizeRealtime` | integer | `3` | Beam search width for the realtime model (min 1) |
| `initialPrompt` | string | `""` | Text prompt to bias main model vocabulary; not emitted in output |
| `initialPromptRealtime` | string | `""` | Text prompt to bias realtime model vocabulary; not emitted in output |

## ONNX Quantization

`onnxQuantization` is the actual precision lever. `""` keeps the model's default precision. Availability is per-model — the model picker only shows the quantizations the selected model supports:

- Most Whisper models: `""`, `fp16`, `q4`, `bnb4`
- lite-whisper: `""`, `fp16`
- NeMo / GigaAM / Vosk: `""`, `int8`
- `large-v3` and `t-tech/t-one`: only `""`

<Callout title="computeType is legacy" type="warn">
  `computeType` is a CTranslate2-era setting with no effect on the ONNX path. It is persisted only for settings/IPC stability. Use `onnxQuantization` to change precision.
</Callout>

## Beam Size

Beam search explores multiple transcription hypotheses in parallel. Higher values produce more accurate results but take longer.

<Callout title="Beam size tuning" type="info">
  Use a low beam size (1-3) for the realtime model to keep live preview responsive. Use a higher beam size for the main model where accuracy matters more than latency.
</Callout>

## Initial Prompt

The `initialPrompt` field accepts free-form text that biases the model toward specific vocabulary. This is useful for:

- Technical terms the model might otherwise mishear
- Proper nouns (company names, product names, people)
- Domain-specific jargon

The prompt does not appear in the output. It only influences how the model interprets ambiguous audio.
